## The Paradigm Shift

Traditional LLM development focused on **train-time compute**:
- Larger models (more parameters)
- More training data
- Longer training runs
- Better optimization algorithms

Reasoning LLMs add **test-time compute**:
- Extended deliberation during inference
- Multiple reasoning steps
- Self-verification and correction
- Dynamic problem-solving strategies

<Info>
  This shift is similar to the difference between memorizing answers and learning to think through problems step-by-step.
</Info>

## Key Concepts Explored

### Chain-of-Thought (CoT) Reasoning

The foundation of reasoning LLMs:

**Prompting Approach**: Simply asking the model to "think step by step"
- No additional training required
- Works with existing models
- Significantly improves performance on reasoning tasks
- Minimal implementation cost

**Few-Shot CoT**: Providing examples of reasoning chains
- Shows the model what good reasoning looks like
- Teaches problem decomposition
- Demonstrates verification steps

<Note>
  Chain-of-thought prompting (covered in Chapter 6) is the starting point. This guide shows how to build reasoning directly into model training.
</Note>

### Process vs. Outcome Supervision

Two approaches to training reasoning models:

**Outcome Supervision**:
- Reward correct final answers
- Simpler to implement
- Doesn't care about the reasoning process
- May lead to "lucky guesses"

**Process Supervision**:
- Reward each correct reasoning step
- More expensive (requires step-level labels)
- Encourages robust reasoning
- Better generalization to new problems

### Test-Time Compute Scaling

The guide illustrates a fascinating property:

- More inference steps → Better performance
- Diminishing returns but consistent improvement
- Tradeoff between latency and accuracy
- Dynamic compute allocation based on problem difficulty

<Info>
  Unlike standard models where inference cost is fixed, reasoning models let you trade compute for accuracy at inference time.
</Info>

## Training Reasoning Models

The guide covers several training approaches:

### Supervised Fine-Tuning on Reasoning Data

1. **Collect reasoning traces**: Human-written step-by-step solutions
2. **Fine-tune on traces**: Train model to generate reasoning chains
3. **Evaluate on reasoning tasks**: Test chain-of-thought quality

### Reinforcement Learning from Human Feedback (RLHF)

- **Reward model**: Trained to score reasoning quality
- **Policy optimization**: Use RL to maximize rewards
- **Exploration**: Discover new reasoning strategies
- **Alignment**: Ensure reasoning aligns with human preferences

### Self-Training Approaches

- **Generate solutions**: Model produces reasoning chains
- **Verify correctness**: Automatic or manual verification
- **Filter and retrain**: Keep good examples, iterate
- **Bootstrap improvement**: Model learns from its own best outputs

## Reasoning Techniques

The guide explores various reasoning strategies:

### Tree-of-Thoughts
- Branch into multiple reasoning paths
- Explore alternatives in parallel
- Backtrack from dead ends
- Select the most promising path

### Self-Verification
- Check reasoning steps for errors
- Validate final answer
- Regenerate if inconsistencies found
- Improves reliability

### Self-Consistency
- Generate multiple reasoning chains
- Take majority vote over answers
- More robust than single chain
- Trades compute for reliability

<Note>
  These techniques can be combined: use tree-of-thoughts for complex problems, self-consistency for reliable answers, and self-verification for correctness.
</Note>
