## The Architecture

DeepSeek-R1 uses a Mixture of Experts (MoE) architecture:

### Scale and Efficiency

- **671 billion total parameters**
- **256 experts** in each MoE layer
- **8 experts activated** per token
- **~37 billion active parameters** per forward pass

<Note>
  The MoE architecture means DeepSeek-R1 has the capacity of a 671B model while using only ~37B parameters per token—similar inference cost to much smaller models.
</Note>

### Why MoE for Reasoning?

- **Specialization**: Different experts handle different reasoning types
- **Efficiency**: Massive capacity without massive compute
- **Scalability**: Can add more experts as needed
- **Performance**: Better than dense models at same active parameter count

## The Training Process

DeepSeek-R1's training is what makes it special:

### Stage 1: Foundation Model

Start with a strong base model:
- Pre-trained on diverse data
- Standard language modeling objective
- No reasoning-specific training yet

### Stage 2: Reasoning Data Collection

Create high-quality reasoning examples:
- Mathematical problems with solutions
- Code challenges with step-by-step solutions
- Scientific reasoning tasks
- Multi-hop logical problems

### Stage 3: Supervised Fine-Tuning

Train on reasoning traces:
- Learn to generate step-by-step solutions
- Develop chain-of-thought capabilities
- Practice self-verification

### Stage 4: Rule-Based Verification

The innovation that sets DeepSeek-R1 apart:

**Automated Verifiers**: Systems that check reasoning quality
- **Code compilation**: Does generated code actually run?
- **Mathematical validity**: Are equations correctly formed?
- **Logical consistency**: Do conclusions follow from premises?
- **Format compliance**: Does output match expected structure?

<Info>
  Rule-based verifiers provide automatic, reliable feedback signals without requiring human labeling for every training example.
</Info>

### Stage 5: Reinforcement Learning

Optimize based on verifier feedback:
- Generate reasoning chains
- Verify correctness automatically
- Reward correct reasoning steps
- Update model via RL
- Repeat to improve

### Stage 6: Self-Improvement

The model teaches itself:
- Generate new problems
- Solve them with reasoning
- Verify solutions
- Train on high-quality self-generated data
- Bootstrap to higher performance

## Key Innovations

### Rule-Based Verifiers

The guide provides detailed visualizations of verification:

**For Code**:
1. Model generates code solution
2. Verifier attempts to compile/execute
3. Test cases validate correctness
4. Success/failure becomes training signal

**For Math**:
1. Model generates mathematical reasoning
2. Verifier checks symbolic manipulation
3. Final answer validated against ground truth
4. Step-level correctness assessed

<Note>
  Verifiers provide dense feedback signals that accelerate learning and improve reliability.
</Note>

### Multi-Turn Reasoning

DeepSeek-R1 can engage in extended reasoning:
- Break complex problems into sub-problems
- Solve incrementally
- Self-correct when errors detected
- Maintain consistency across reasoning chain

### Long-Context Reasoning

The model handles extended contexts:
- Process long problem descriptions
- Maintain reasoning state over many steps
- Reference earlier parts of reasoning chain
- Build complex arguments incrementally
