# Text Clustering and Topic Modeling

Text clustering groups similar documents together without predefined labels, making it ideal for exploratory analysis and discovering themes in large text collections. This chapter covers the complete pipeline from embeddings to topic modeling using modern techniques.

<Note>
This chapter uses **44,949 ArXiv abstracts** from the Computation and Language category to demonstrate clustering and topic modeling at scale.
</Note>

## Overview

Text clustering and topic modeling help you:

- **Discover themes** in large document collections
- **Organize documents** without manual labeling
- **Understand dataset structure** through visualization
- **Extract interpretable topics** with representative keywords
- **Track topic evolution** over time

## Dataset Setup

Load the ArXiv NLP papers dataset:

```python
from datasets import load_dataset

# Load ArXiv abstracts (Computation and Language)
dataset = load_dataset("maartengr/arxiv_nlp")["train"]

# Extract metadata
abstracts = list(dataset["Abstracts"])
titles = list(dataset["Titles"])

print(f"Total documents: {len(abstracts)}")  # 44,949 abstracts
```

## The Common Pipeline for Text Clustering

Text clustering typically follows three key steps:

<Steps>

<Step title="1. Embed Documents">
Convert text to dense vector representations
</Step>

<Step title="2. Reduce Dimensionality">
Compress embeddings to lower dimensions while preserving structure
</Step>

<Step title="3. Cluster Documents">
Group similar documents using density-based clustering
</Step>

</Steps>

Let's implement each step in detail.

## Step 1: Embedding Documents

Create semantic embeddings using a sentence transformer:

```python
from sentence_transformers import SentenceTransformer

# Load a lightweight but powerful embedding model
embedding_model = SentenceTransformer('thenlper/gte-small')

# Create embeddings for all abstracts
embeddings = embedding_model.encode(abstracts, show_progress_bar=True)

print(embeddings.shape)  # (44949, 384)
# 44,949 documents × 384-dimensional embeddings
```

<Tip>
**Model Selection**: `thenlper/gte-small` provides a good balance between quality and speed. For better quality, try `sentence-transformers/all-mpnet-base-v2` (768 dims). For production speed, use `sentence-transformers/all-MiniLM-L6-v2` (384 dims).
</Tip>

## Step 2: Dimensionality Reduction with UMAP

Reduce 384 dimensions to 5 dimensions using UMAP:

```python
from umap import UMAP

# Configure UMAP for dimensionality reduction
umap_model = UMAP(
    n_components=5,        # Reduce to 5 dimensions
    min_dist=0.0,          # Controls how tightly points cluster
    metric='cosine',       # Use cosine distance for text
    random_state=42        # Reproducibility
)

# Fit and transform embeddings
reduced_embeddings = umap_model.fit_transform(embeddings)

print(reduced_embeddings.shape)  # (44949, 5)
```

<Note>
**Why 5 dimensions?** UMAP to 5D preserves local and global structure while making clustering more effective. For visualization, we'll reduce to 2D separately.
</Note>

## Step 3: Clustering with HDBSCAN

Cluster the reduced embeddings using HDBSCAN:

```python
from hdbscan import HDBSCAN

# Configure HDBSCAN for density-based clustering
hdbscan_model = HDBSCAN(
    min_cluster_size=50,              # Minimum documents per cluster
    metric='euclidean',               # Distance metric
    cluster_selection_method='eom'    # Excess of Mass method
)

# Fit and extract cluster labels
hdbscan_model.fit(reduced_embeddings)
clusters = hdbscan_model.labels_

# Count unique clusters (including -1 for outliers)
print(f"Number of clusters: {len(set(clusters))}")  # 156 clusters
print(f"Outliers: {sum(clusters == -1)}")  # Documents not assigned to clusters
```

<Warning>
**Outlier Cluster**: HDBSCAN assigns label `-1` to outliers - documents that don't fit well into any cluster. This is a feature, not a bug!
</Warning>

### Visualizing Clusters

Reduce to 2D for visualization:

```python
import pandas as pd
import matplotlib.pyplot as plt

# Reduce embeddings to 2D for visualization
reduced_embeddings_2d = UMAP(
    n_components=2, 
    min_dist=0.0, 
    metric='cosine', 
    random_state=42
).fit_transform(embeddings)

# Create dataframe
df = pd.DataFrame(reduced_embeddings_2d, columns=["x", "y"])
df["title"] = titles
df["cluster"] = [str(c) for c in clusters]

# Separate outliers from clusters
clusters_df = df.loc[df.cluster != "-1", :]
outliers_df = df.loc[df.cluster == "-1", :]

# Plot
plt.figure(figsize=(12, 8))
plt.scatter(outliers_df.x, outliers_df.y, alpha=0.05, s=2, c="grey", label="Outliers")
plt.scatter(
    clusters_df.x, 
    clusters_df.y, 
    c=clusters_df.cluster.astype(int),
    alpha=0.6, 
    s=2, 
    cmap='tab20b',
    label="Clusters"
)
plt.axis('off')
plt.title('ArXiv NLP Papers Clustering', fontsize=16)
plt.show()
```

## From Clustering to Topic Modeling

Clustering groups documents, but **topic modeling** adds interpretable labels to each cluster using representative keywords.

### BERTopic: Modular Topic Modeling

BERTopic combines all three steps with automatic topic extraction:

```python
from bertopic import BERTopic

# Create BERTopic model with our pre-configured components
topic_model = BERTopic(
    embedding_model=embedding_model,
    umap_model=umap_model,
    hdbscan_model=hdbscan_model,
    verbose=True
)

# Fit the model
topics, probs = topic_model.fit_transform(abstracts, embeddings)
```

## Advanced: Custom Representation Models

BERTopic supports multiple representation models for better topic descriptions:

### KeyBERT-Inspired Representation

Use document embeddings to find the most representative keywords:

```python
from bertopic.representation import KeyBERTInspired

# Create representation model
representation_model = KeyBERTInspired()

# Update topic representations
topic_model.update_topics(
    abstracts, 
    representation_model=representation_model
)
```

### Using LLMs for Topic Labels

Generate human-readable topic labels with ChatGPT:

```python
from bertopic.representation import OpenAI
import openai

# Create OpenAI representation model
client = openai.OpenAI(api_key="YOUR_KEY_HERE")
representation_model = OpenAI(
    client,
    model="gpt-3.5-turbo",
    chat=True,
    nr_docs=5,
    prompt="I have topic with the following documents: [DOCUMENTS]. Based on the information above, create a short topic label in the following format: 'topic: <topic label>'"
)

# Update topics with LLM-generated labels
topic_model.update_topics(
    abstracts,
    representation_model=representation_model
)
```

## Evaluation Metrics

### Cluster Quality Metrics

```python
from sklearn.metrics import silhouette_score, calinski_harabasz_score

# Remove outliers for evaluation
non_outliers = clusters != -1

# Silhouette Score (higher is better, range: -1 to 1)
silhouette = silhouette_score(
    reduced_embeddings[non_outliers], 
    clusters[non_outliers]
)
print(f"Silhouette Score: {silhouette:.3f}")

# Calinski-Harabasz Score (higher is better)
ch_score = calinski_harabasz_score(
    reduced_embeddings[non_outliers], 
    clusters[non_outliers]
)
print(f"Calinski-Harabasz Score: {ch_score:.2f}")
```

### Topic Coherence

Measure topic interpretability:

```python
from gensim.models import CoherenceModel
from gensim.corpora import Dictionary

# Prepare documents for coherence calculation
tokenized_docs = [doc.lower().split() for doc in abstracts]
dictionary = Dictionary(tokenized_docs)

# Get topics as word lists
topics_words = []
for topic_id in range(len(set(clusters)) - 1):
    words = [word for word, _ in topic_model.get_topic(topic_id)]
    topics_words.append(words)

# Calculate coherence
coherence_model = CoherenceModel(
    topics=topics_words,
    texts=tokenized_docs,
    dictionary=dictionary,
    coherence='c_v'
)
coherence = coherence_model.get_coherence()
print(f"Topic Coherence (C_V): {coherence:.3f}")
```

## Best Practices

<CardGroup cols={2}>

<Card title="Embedding Selection" icon="vector-square">
- Use domain-specific embeddings when available
- Balance embedding dimension with dataset size
- Consider multilingual models for non-English text
- Test multiple embedding models for your use case
</Card>

<Card title="Hyperparameter Tuning" icon="sliders">
- `min_cluster_size`: Increase for larger, more general topics
- `n_components`: 5D works well for most cases
- `min_dist` (UMAP): 0.0 for tight clusters, 0.1+ for looser
- `cluster_selection_method`: Try 'eom' and 'leaf'
</Card>

<Card title="Handling Outliers" icon="filter">
- Outliers (topic -1) are normal - often 20-30% of documents
- Reduce `min_cluster_size` to assign more documents
- Use `reduce_outliers()` method to reassign outliers
- Consider outliers as "diverse" or "multi-topic" documents
</Card>

<Card title="Interpretability" icon="eye">
- Use representation models for better topic descriptions
- Examine representative documents, not just keywords
- Create topic labels manually for critical applications
- Validate topics with domain experts
</Card>

</CardGroup>

## Performance Tips

<Steps>

<Step title="Pre-compute Embeddings">
Generate and save embeddings once to speed up experimentation:

```python
import numpy as np

# Save embeddings
np.save('embeddings.npy', embeddings)

# Load later
embeddings = np.load('embeddings.npy')
```
</Step>

<Step title="Use GPU Acceleration">
Speed up embedding generation with GPU:

```python
model = SentenceTransformer('thenlper/gte-small', device='cuda')
embeddings = model.encode(texts, show_progress_bar=True, batch_size=32)
```
</Step>

<Step title="Batch Processing">
For very large datasets, process in batches:

```python
from tqdm import tqdm

batch_size = 1000
all_embeddings = []

for i in tqdm(range(0, len(abstracts), batch_size)):
    batch = abstracts[i:i+batch_size]
    batch_embeddings = model.encode(batch)
    all_embeddings.append(batch_embeddings)

embeddings = np.vstack(all_embeddings)
```
</Step>

</Steps>
